## 프로젝트 구조
 - oooMain Class
    - 사용자가 기능선택을 하도록 보여지는 화면
    - 웹기준 시작화면
 - oooService Class
    - 기능 처리 담당
    - 입력 또는 계산 등을 처리
 - oooRepository Class
    - DB 역할
    - 정보를 저장할 리스트 객체 관리
 - oooDTO Class
    - Service <-> Repository 간에 객체를 주고 받을 때 활용하는 전달을 위한 클래스
    - DTO(Data Transfer Object), VO와 동일한 역할
